# FreshTrack Context Review Prompt

Проанализируй изменения в контексте всего Merge Request для проекта **<<project_name>>**.

## О проекте
<<project_name>> — это система управления запасами для отелей с функциями:
- Учёт партий товаров с датами истечения срока годности
- FIFO-списание (First In, First Out)
- Уведомления о истекающих сроках
- Мультитенантность (изоляция данных между отелями)
- RBAC (Role-Based Access Control)

## Технологии
- **Frontend**: React 18, Vite, TailwindCSS, Zustand
- **Backend**: Express.js, PostgreSQL
- **Auth**: JWT tokens
- **i18n**: 3 языка (ru, en, kk)

## Что анализировать

### 1. Архитектурная целостность
- Соответствуют ли изменения паттернам проекта?
- Нет ли "костылей" или обходных решений?
- Правильно ли разделены слои (API, Service, Database)?

### 2. Взаимодействие компонентов
- Как изменения влияют на другие части системы?
- Нет ли конфликтов между frontend и backend?
- Корректны ли API контракты?

### 3. Безопасность в контексте
- Правильно ли применяются middleware (auth, permissions)?
- Нет ли утечки данных между tenant'ами?
- Проверяются ли права доступа на всех уровнях?

### 4. Бизнес-логика
- Корректна ли логика FIFO?
- Правильно ли рассчитываются статусы (expired, warning, critical)?
- Соответствуют ли изменения бизнес-требованиям?

## Формат ответа

Дай структурированный обзор:

1. **Общая оценка** (1-5 звёзд)
2. **Резюме изменений** (2-3 предложения)
3. **Найденные проблемы** (если есть)
4. **Рекомендации** (что улучшить)
5. **Положительные моменты** (что хорошо)
